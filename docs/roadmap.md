# Roadmap

Rex is actively evolving to support an ecosystem capable of completely replacing the modern web stack.

The immediate priorities for development include:

- **Rex State**
  Reactive variables with zero boilerplate, automatically syncing variables with UI states.

- **Rex Components**
  Define and package reusable, modular UI pieces using native syntax: `define Button(label, color):`.

- **Rex Animate**
  Declarative, native-speed animations using plain English logic: `animate: slide in from left over 0.3 seconds`.

- **Rex Web**
  A compiler target for WebAssembly, allowing Rex applications to run performantly within traditional browser environments when required.

- **Rex Mobile**
  First-class compiler targets for iOS and Android, ensuring the same `.rex` file executes natively across desktop and mobile.

- **Rex Server**
  Server-side logic written entirely in Rex, managing HTTP connections, routing, and databases using the clean Rex syntax.

- **Rex Studio**
  A dedicated visual IDE where designers can manipulate UI components graphically while writing `.rex` code, instantly seeing the output.

- **The Rex Package Registry**
  A unified community platform for sharing reusable components, design themes, logic modules, and icons written in Rex.
